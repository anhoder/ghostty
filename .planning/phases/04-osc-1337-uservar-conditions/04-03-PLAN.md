Update `matchesCondition` function's `.var_` case:

Current implementation uses `std.mem.eql` for exact match only. Enhance to support glob:

1. Check if condition value contains wildcards:
   ```zig
   const has_wildcards = std.mem.indexOfAny(u8, cond.value, "*?") != null;
   ```

2. If no wildcards, use existing exact match (fast path):
   ```zig
   if (!has_wildcards) {
       return std.mem.eql(u8, actual_value, cond.value);
   }
   ```

3. If wildcards present, use glob matching:
   ```zig
   return std.mem.match(u8, actual_value, cond.value);
   ```

Note: Use `std.mem.match` if available, or `std.fs.path.match` as fallback. Research confirms stdlib has glob support.

Keep the common case (exact match) fast by checking for wildcards first.
