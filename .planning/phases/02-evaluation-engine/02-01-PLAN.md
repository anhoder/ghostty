1. Add `runtime_context` field to the Surface struct, near the `keyboard` field (around line 93):
   ```zig

   ```
   Default value handles initialization — no change to `Surface.init` needed.

2. Update `maybeHandleBinding` (line 2825 entry: block). The root-set path at line 2874-2876 currently does:
   ```zig
   break :entry self.config.keybind.set.getEvent(event) orelse return null;
   ```
   This produces a `Set.Entry` (HashMap pointer). Replace with a path that uses `getEventConditional` and bypasses the `entry:` block entirely for the root set:

   Restructure the `entry:` block so that:
   - Sequence set path: unchanged (`set.getEvent(event)` → `break :entry`)
   - Table stack path: unchanged (`table.set.getEvent(event)` → `break :entry`)
   - Root set path: instead of `break :entry`, call `getEventConditional` and handle the `ConditionalResult` directly

   The cleanest approach: after the table stack loop, instead of `break :entry self.config.keybind.set.getEvent(event)`, do:
   ```zig
   // Root set: use conditional evaluation
   const cond_result = self.config.keybind.set.getEventConditional(
       event,
       &self.runtime_context,
   ) orelse return null;
   // ConditionalResult → GenericLeaf directly (skip the entry: block and leader switch)
   // ConditionalResult only returns for leaf bindings (getConditional returns null for leader/leaf_chained),
   // so we can construct GenericLeaf directly:
   const leaf: input.Binding.Set.GenericLeaf = .{
       .flags = cond_result.flags,
       .actions = .{ .single = .{cond_result.action} },
   };
   ```
   Then jump past the `entry:` block and the `switch (entry.value_ptr.*)` to the `consumed:` block at line 2910. Use a labeled block or restructure the flow so the root-set path skips the leader/leaf switch (since ConditionalResult never returns leaders).

   IMPORTANT: The sequence and table paths still need the `entry:` block and the `switch (entry.value_ptr.*)` for leader handling. Only the root-set path changes. The simplest restructure:
   - Keep the `entry:` block for sequence + table paths
   - Add an early-return/goto for the root-set conditional path that constructs GenericLeaf directly and jumps to the `consumed:` block

3. Update `keyEventIsBinding` (line 2576 entry: block). Same pattern — only the root-set lookup at line 2592 changes:
   ```zig
   // Before:
   break :entry self.config.keybind.set.getEvent(event) orelse return null;
   // After:
   const cond_result = self.config.keybind.set.getEventConditional(
       event,
       &self.runtime_context,
   ) orelse return null;
   return cond_result.flags;
   ```
   This is simpler because `keyEventIsBinding` only returns `Flags`, not a full entry. The conditional path can return `cond_result.flags` directly without going through the entry block.

4. Do NOT touch:
   - Sequence set lookups (line 2829: `set.getEvent(event)`)
   - Table stack lookups (line 2859: `table.set.getEvent(event)`)
   - `last_trigger` tracking logic
   - Any other Surface methods
